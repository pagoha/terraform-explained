# terraform-explained

A zero-to-hero guide to Terraform using AWS, with safe, step-by-step working demos.

This repository is designed for anyone—from complete beginners to experienced IT professionals—who wants to learn Terraform in a structured, hands-on way. By following the demos and reading the Insights sections, learners will progress from setting up Terraform locally to confidently deploying AWS resources and managing more complex infrastructure.

---

## Repo Structure

The content is organized into **phases**, each building on the previous:

* **Phase 0 – Setup & Orientation:** Install Terraform, configure AWS, and run your first local resource.
* **Phase 1 – Single AWS Resource:** Create and manage basic AWS resources.
* **Phase 2 – Variables & Outputs:** Learn how to parameterize and modularize your infrastructure.
* **Phase 3 – Multiple Resources & Dependencies:** Manage interdependent AWS resources.
* **Phase 4 – Modules:** Introduce reusable infrastructure patterns.
* **Phase 5 – Remote State & Backends:** Safely manage state for collaboration.
* **Phase 6 – Environments / Workspaces:** Separate dev/test/prod configurations.
* **Phase 7 – Refactoring & Maintenance:** Rename, import, and maintain resources.
* **Phase 8 – Hero Challenges:** Advanced multi-resource examples, error handling, and CI/CD integration.

---

## Quick Start

Follow this simple workflow to get started:

```text
Clone repo → phase-0-setup-orientation → run first demo → read Insights
           ↓
     Continue Phase 0 → ensure Terraform and AWS configured
           ↓
       Progress sequentially through Phases 1–8
           ↓
    Become confident deploying real AWS infrastructure with Terraform
```

**Tips:**

* Read each **Overview** first to understand the concept.
* Review **Code Example** and run it in your own AWS environment.
* Compare results to **Expected Output**.
* Reflect on **Insights** to learn best practices, common mistakes, and next steps.
