# 4-4 Module Best Practices

## Overview

Learn best practices for structuring Terraform modules.

## Key Guidelines

* Keep modules small and focused
* Use variables for all configurable options
* Use outputs to expose useful resource info
* Avoid hardcoding values
* Include README inside each module explaining usage

## Insights

* **Why this demo exists:** Ensures maintainable and reusable Terraform code.
* **Reflection / next steps:** Apply these principles to Phase 5 (Environments & Workspaces) and Phase 6 (Remote State & Collaboration).
